# 项目访问说明

## 重要说明

### 1. **不需要运行额外的服务**

对于构建后的静态项目（如 React、Vue 等前端项目），**不需要运行额外的服务**。

- ✅ **构建后的项目**：只需要 `npm run build` 生成 `dist` 目录，然后通过 Express 静态文件服务直接访问
- ❌ **不需要**：`npm start` 或 `npm run dev`（这些是开发服务器，用于开发环境）

### 2. **访问方式**

#### 方式一：通过目录访问（推荐）
```
http://localhost:3000/us-cryptography-service-platform/dist/
```
Express 会自动查找 `index.html` 并返回。

#### 方式二：直接访问文件
```
http://localhost:3000/us-cryptography-service-platform/dist/index.html
```

### 3. **资源路径配置**

为了确保资源文件（JS、CSS）能正确加载，需要在 `vite.config.js` 中设置：

```javascript
export default defineConfig({
  plugins: [react()],
  base: './', // 使用相对路径，这样资源可以在任何子目录下正确加载
})
```

这样构建后的 HTML 文件中的资源路径会是相对路径（如 `./assets/index.js`），而不是绝对路径（如 `/assets/index.js`）。

### 4. **完整流程**

1. **Git 同步**：从 Git 仓库同步代码
2. **安装依赖**：`npm install`（自动执行）
3. **构建项目**：`npm run build`（自动执行）
4. **访问项目**：通过 `http://localhost:3000/项目名/dist/` 访问

### 5. **为什么不需要额外服务？**

- **开发环境**：使用 `npm run dev` 启动开发服务器（如 Vite Dev Server），提供热重载等功能
- **生产环境**：使用 `npm run build` 构建静态文件，然后通过 Web 服务器（如 Express、Nginx）提供静态文件服务

我们的系统就是 Web 服务器（Express），所以可以直接提供静态文件服务，**不需要运行项目的开发服务器**。

### 6. **当前项目状态**

- ✅ 项目已构建：`dist` 目录已生成
- ✅ 资源路径已修复：使用相对路径
- ✅ 可以访问：`http://localhost:3000/us-cryptography-service-platform/dist/`

### 7. **如果页面显示空白**

可能的原因：
1. 浏览器缓存：清除浏览器缓存或使用无痕模式
2. 资源路径问题：检查浏览器控制台的网络请求，看是否有 404 错误
3. JavaScript 错误：检查浏览器控制台的错误信息

### 8. **验证访问**

在浏览器中打开：
```
http://localhost:3000/us-cryptography-service-platform/dist/
```

应该能看到完整的应用界面。
